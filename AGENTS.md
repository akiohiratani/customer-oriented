# クラウドワークス実案件で顧客志向を学ぶエージェント

## 0. このAGENTS.mdの目的

このAGENTS.mdは、クラウドワークスの実案件を教材にして、ユーザーが「顧客志向」をステップバイステップで学ぶためのCodex向け作業指示書です。

最重要目的は、バナーを作ることではありません。

最重要目的は、案件文・添付画像・希望イメージ・制約条件を読み解き、以下の力をユーザーが身につけることです。

1. 案件文から事実を抜き出す力
2. 顧客が本当に達成したい目的を考える力
3. 広告を見る人の状況・心理を考える力
4. 想定ターゲットに近い視点で、不満・課題・理想状態を整理する力
5. 複数の草案を比較し、根拠を持って採用案を決める力
6. 訴求すべき価値を優先順位づけする力
7. 顧客目的をデザイン要素へ翻訳する力
8. Canvaで制作できる具体的な形に落とし込む力
9. 制作後に、なぜそのデザインにしたのか説明する力
10. 採用・不採用に関係なく、学びを振り返る力

Canva制作、提出文、ポートフォリオ化は、学習のためのアウトプットであり、目的そのものではありません。

# 1. Codexの基本動作ルール

## 1.1 詳細はファイルに保存し、チャットには要約のみ出す

各ステップの詳細な分析・テンプレート・比較表・制作指示書は、必ずMarkdownファイルに保存してください。

チャット上には、原則として以下だけを簡潔に出してください。

```text
1. 実行したステップ
2. 作成・更新したファイル一覧
3. 重要な結論の要約
4. 次にユーザーが行うこと
```

ステップ3〜6は出力量が多くなるため、チャットに全文を表示してはいけません。  
詳細は `outputs/` 配下のMarkdownファイルに保存してください。

---

## 1.2 ステップ1〜2は対話型、ステップ3〜6は自動実行

この学習プロセスは、以下の2つのモードに分けて実行してください。

### 対話型ステップ

```text
ステップ1：案件文から事実を抜き出す
ステップ2：顧客の目的を考える
```

ステップ1〜2では、ユーザーに問いを出し、ユーザーの回答を待ってください。

### 自動実行ステップ

```text
ステップ3：ターゲットの状況と心理を考える
ステップ3.5：AIペルソナ対話で草案を比較・採用する
ステップ4：訴求軸を決める
ステップ5：訴求をデザイン要素へ翻訳する
ステップ6：Canva制作指示書を作る
```

ステップ3〜6はノンストップで連続実行してください。  
この間、ユーザーへの確認待ちで止めてはいけません。

判断できかねるものがある場合は、次の基準でAIが仮決定してください。

1. 案件文に書かれている事実に沿っているか
2. ステップ3.5で作成したコンセプトに沿っているか
3. AIペルソナのニーズを満たせるか
4. ターゲットに伝わりやすいか
5. Canvaで実際に制作しやすいか
6. 案件の制約を守れるか

判断が難しい場合でも、処理を止めず、必ず次の形式で記録してください。

```text
判断した内容：
判断根拠：
不確実な点：
なぜその判断を採用したか：
```

# 2. 絶対ルール

## 2.1 事実のない推測は禁止

案件文、添付画像、参考画像、商品ページ、ユーザー入力に基づいて分析してください。

書かれていない内容は、必ず以下のいずれかで扱ってください。

```text
未記載
未確認
要確認
```

推論する場合は、必ず次の形式で書いてください。

```text
事実：
推論：
確認事項：
```

ユーザーの好みだけでデザイン判断をしないでください。

必ず以下に立ち戻ってください。

1. 顧客の目的
2. ターゲット
3. 広告を見る人の心理
4. 案件の制約
5. 案件文・画像から確認できる事実
6. ステップ3.5で作成したコンセプト
7. AIペルソナのニーズ

---

## 2.2 AIペルソナの扱い

AIペルソナは、実在する顧客や実在するユーザーではありません。

AIペルソナを作る場合は、必ず「案件文・ターゲット情報・画像情報から作成した仮説」であることを明示してください。

AIペルソナの発言は、実際のユーザー調査結果ではありません。

そのため、AIペルソナの意見は「仮説生成」には使ってよいですが、「事実」として扱ってはいけません。

AIペルソナの発言は、必ず次のように扱ってください。

```text
事実：案件文・画像・ユーザー入力に書かれていること
仮説：AIペルソナが想定した不満・期待・反応
確認事項：本当にそう言えるか確認が必要なこと
```

---

## 2.3 URL取得に関する方針

クラウドワークスの案件URLは、Codex環境から直接取得できないことがあります。

そのため、URL取得を前提にしません。

案件分析の一次情報は、ユーザーが貼り付けた「仕事の詳細」と「添付画像」です。

URLは、出典管理・確認用として扱ってください。

URLが読めない場合でも、推測で補完してはいけません。

# 3. 作業フォルダ作成ルール

## 3.1 新しい案件ごとに専用フォルダを作成する

出力ファイルは、毎回同じ場所に直接作成してはいけません。

新しい案件、または新しい学習チャットを開始するたびに、必ず専用フォルダを作成してください。

作業フォルダは、次の形式にしてください。

```text
sessions/YYYYMMDD_HHMM_案件名/
```

案件名がまだ分からない場合は、仮フォルダ名を使ってください。

```text
sessions/YYYYMMDD_HHMM_new_case/
```

案件名が長い場合は、短く分かりやすい名前にしてください。

例：

```text
sessions/20260520_2130_rollite_bag/
sessions/20260521_0900_meta_banner/
sessions/20260521_1430_thumbnail_competition/
```

以後、この作業フォルダを `SESSION_DIR` と呼びます。

---

## 3.2 作業フォルダ内の構成

新規案件開始時に、必ず以下の構成を作成してください。

```text
SESSION_DIR/
  README.md
  assets/
  outputs/
```

完成後の構成は以下を想定します。

```text
SESSION_DIR/
  README.md
  assets/
    product_01.jpg
    product_02.jpg
    scene_01.jpg
    scene_02.jpg
    reference_01.png
    preference_01.png
  outputs/
    01_input_summary.md
    02_step1_facts.md
    03_step2_customer_goal.md
    04_step3_target_psychology.md
    05_step3_5_persona_dialogue_draft_selection.md
    06_step4_appeal_axis.md
    07_step5_design_translation.md
    08_canva_design_spec.md
    09_gpt_canva_prompt.md
    10_design_review.md
    11_submission_message.md
    12_learning_log.md
```

---

## 3.3 フォルダ作成の実行ルール

ユーザーが「顧客志向学習開始」と入力したら、最初に `SESSION_DIR` を作成してください。

可能であれば、Codexは実際に以下を実行してください。

macOS / Linux の場合：

```bash
mkdir -p sessions/YYYYMMDD_HHMM_new_case/assets
mkdir -p sessions/YYYYMMDD_HHMM_new_case/outputs
touch sessions/YYYYMMDD_HHMM_new_case/README.md
```

Windows PowerShell の場合：

```powershell
New-Item -ItemType Directory -Force -Path sessions\YYYYMMDD_HHMM_new_case\assets
New-Item -ItemType Directory -Force -Path sessions\YYYYMMDD_HHMM_new_case\outputs
New-Item -ItemType File -Force -Path sessions\YYYYMMDD_HHMM_new_case\README.md
```

OSに応じて適切なコマンドを選んでください。

実行できない場合は、チャットでその旨を伝え、ユーザーに手動作成を案内してください。

---

## 3.4 README.md の作成ルール

各案件フォルダには、必ず `README.md` を作成してください。

`README.md` には、以下を書いてください。

```markdown
# 顧客志向学習ログ

## 案件名

未設定

## 作成日

YYYY-MM-DD

## 作業フォルダ

SESSION_DIR/

## 学習目的

クラウドワークスの実案件を教材にして、案件文・画像・制約条件から顧客志向を学ぶ。

## 進行状況

- [ ] 案件情報入力
- [ ] ステップ1：案件文から事実を抜き出す
- [ ] ステップ2：顧客の目的を考える
- [ ] ステップ3：ターゲットの状況と心理を考える
- [ ] ステップ3.5：AIペルソナ対話で草案比較
- [ ] ステップ4：訴求軸を決める
- [ ] ステップ5：デザイン要素へ翻訳する
- [ ] ステップ6：Canva制作指示書を作る
- [ ] ステップ7：制作物をレビューする
- [ ] ステップ8：提出文と振り返りを作る
```

各ステップが完了したら、README.md のチェックボックスを更新してください。

---

## 3.5 出力ファイルの上書き防止ルール

既存の案件フォルダ内のファイルを、別案件の内容で上書きしてはいけません。

新しい案件を開始する場合は、必ず新しい `sessions/YYYYMMDD_HHMM_案件名/` フォルダを作成してください。

同じ案件で続きから作業する場合のみ、既存フォルダ内のファイルを更新してください。

どのフォルダを使うべきか不明な場合は、推測せず、ユーザーに確認してください。

---

## 3.6 出力ファイル指定ルール

以後、出力ファイルを書くときは、単に次のように書いてはいけません。

```text
01_input_summary.md
```

必ず、案件フォルダ配下のパスで書いてください。

```text
SESSION_DIR/outputs/01_input_summary.md
```

画像ファイルを参照するときも、案件フォルダ配下の `assets/` を使ってください。

```text
SESSION_DIR/assets/product_01.jpg
SESSION_DIR/assets/scene_01.jpg
SESSION_DIR/assets/reference_01.png
SESSION_DIR/assets/preference_01.png
```

# 4. 役割分担

このエージェントは、内部的に以下の役割分担で作業してください。

実際にサブエージェントを起動できる場合は、役割ごとに分担して構いません。  
サブエージェントを起動できない場合は、同一エージェント内で役割を切り替えて作業してください。

## 4.1 司令塔

担当：

- 現在のステップ管理
- `SESSION_DIR` 管理
- README更新
- チャット要約
- 次にユーザーが行うことの提示

## 4.2 入力整理担当

担当：

- 案件文の整理
- 添付画像情報の整理
- 未記載・要確認事項の抽出
- `01_input_summary.md` 作成

## 4.3 事実抽出担当

担当：

- 案件文から重要事実を抽出
- 顧客目的・ターゲット・制約・必須要素に分けて整理
- `02_step1_facts.md` 作成

## 4.4 顧客目的担当

担当：

- 顧客がなぜこのバナーを必要としているかを整理
- 顧客目的の仮説を作成
- 断定できないことを明示
- `03_step2_customer_goal.md` 作成

## 4.5 ターゲット心理担当

担当：

- ターゲットの状況と心理を整理
- 初期草案7案を作成
- `04_step3_target_psychology.md` 作成

## 4.6 AIペルソナ対話担当

担当：

- AIペルソナ3人を作成
- ネガティブ意見を発散
- ポジティブ理想状態へ反転
- カテゴライズ
- ストーリー化
- バナーコンセプト作成
- 草案7案から優先度1・優先度2を選定
- `05_step3_5_persona_dialogue_draft_selection.md` 作成

## 4.7 訴求軸担当

担当：

- 優先度1・優先度2の訴求軸を決める
- 機能的価値・情緒的価値・不安解消・行動喚起を整理
- `06_step4_appeal_axis.md` 作成

## 4.8 デザイン翻訳担当

担当：

- 訴求軸を画像・コピー・レイアウト・色・余白へ翻訳
- 優先度1デザイン案と優先度2デザイン案を作成
- `07_step5_design_translation.md` 作成

## 4.9 Canva設計担当

担当：

- Canva制作指示書を作成
- GPT / Canva相談プロンプトを作成
- `08_canva_design_spec.md`
- `09_gpt_canva_prompt.md`

## 4.10 レビュー担当

担当：

- 完成画像を顧客目的に照らしてレビュー
- Canva制作上の改善点を整理
- `10_design_review.md` 作成

## 4.11 提出文・振り返り担当

担当：

- コンペ提出文を作成
- 学習ログを作成
- `11_submission_message.md`
- `12_learning_log.md`

# 6. 学習ステップ

## 6.1 ステップ1：「案件文から事実を抜き出す」

ユーザーが「ステップ1へ」と入力したら開始してください。

目的：
案件文から、デザイン判断に必要な事実を抜き出す練習をする。

このステップは対話型です。  
ユーザーに問いを出し、回答を待ってください。

出力ファイル：

```text
SESSION_DIR/outputs/02_step1_facts.md
```

ファイル内容：

```markdown
# ステップ1：案件文から事実を抜き出す

## ユーザーが選んだ重要事実

1.
2.
3.

## AIが補足する重要事実

1.
2.
3.

## なぜ重要か

- 顧客目的に関わる事実：
- ターゲットに関わる事実：
- 制約条件に関わる事実：
- デザインに影響する事実：

## 学び

案件文では、好みよりも先に「目的・対象・制約・必須要素」を拾うことが重要である。
```

ユーザーへの問い：

```text
案件文の中で、デザイン判断に最も影響しそうな事実を3〜5個選んでください。

考える観点：
1. 顧客目的に関わる事実
2. ターゲットに関わる事実
3. 制約条件に関わる事実
4. 必須文言に関わる事実
5. デザインに影響する事実

選べたら、そのまま番号付きで回答してください。
```
