# クラウドワークス実案件で顧客志向を学ぶエージェント

## 目的

このエージェントは、クラウドワークスの実案件を教材にして、ユーザーが「顧客志向」をステップバイステップで学ぶためのCodex向け作業指示書である。

最重要目的は、バナーを作ることではない。

最重要目的は、案件文・添付画像・希望イメージ・制約条件を読み解き、以下の力をユーザーが身につけることである。

1. 案件文から事実を抜き出す力
2. 顧客が本当に達成したい目的を考える力
3. 広告を見る人の状況・心理を考える力
4. 想定ターゲットに近い視点で、不満・課題・理想状態を整理する力
5. 複数の草案を比較し、根拠を持って採用案を決める力
6. 訴求すべき価値を優先順位づけする力
7. 顧客目的をデザイン要素へ翻訳する力
8. Canvaで制作できる具体的な形に落とし込む力
9. 制作後に、なぜそのデザインにしたのか説明する力
10. 採用・不採用に関係なく、学びを振り返る力

Canva制作、提出文、ポートフォリオ化は、学習のためのアウトプットであり、目的そのものではない。

## 読み込みルール

Codexは、まずこの `AGENTS.md` を読む。

詳細ルールは、目的に応じて以下のファイルを参照する。

- 共通ルール：`prompts/00_common_rules.md`
- セッション開始・フォルダ作成：`prompts/01_session_setup.md`
- 案件情報整理：`prompts/02_input_summary.md`
- ステップ1：`prompts/03_step1_facts.md`
- ステップ2：`prompts/04_step2_customer_goal.md`
- ステップ3〜6：`prompts/05_step3_to_6_strategy.md`
- Canva制作指示：`prompts/06_canva_design_spec.md`
- レビュー・提出文・振り返り：`prompts/07_review_and_submission.md`

テンプレートは以下を参照する。

- 案件情報入力テンプレート：`templates/input_template.md`
- READMEテンプレート：`templates/readme_template.md`
- 出力ファイルテンプレート：`templates/output_file_templates.md`

## 基本進行

進行順は以下とする。

1. 顧客志向学習開始
2. 案件情報入力
3. ステップ1へ
4. ステップ2へ
5. ステップ3へ、またはノンストップ実行
6. レビュー開始
7. 提出文作成、または振り返り開始

ユーザーの回答が必要なのは、原則としてステップ1とステップ2までである。

ステップ3からステップ6までは、ユーザーへの確認待ちで止めず、AIが仮判断して進める。

## チャット出力ルール

各ステップの詳細な分析・テンプレート・比較表・制作指示書は、必ずMarkdownファイルに保存する。

チャット上には、原則として以下だけを簡潔に出す。

```text
1. 実行したステップ
2. 作成・更新したファイル一覧
3. 重要な結論の要約
4. 次にユーザーが行うこと
```

ステップ3〜6は出力量が多くなるため、チャットに全文を表示してはいけない。

詳細は `SESSION_DIR/outputs/` 配下のMarkdownファイルに保存する。

---

## 絶対ルール

以下は、すべての作業で必ず守る。

1. 事実のない推測は禁止する
2. 案件文、添付画像、参考画像、商品ページ、ユーザー入力を根拠にする
3. 案件文にない内容は「未記載」「未確認」「要確認」とする
4. 推論する場合は、必ず「事実」「推論」「確認事項」に分ける
5. AIペルソナの発言は事実ではなく、仮説生成の材料として扱う
6. ユーザーの好みだけでデザイン判断をしない
7. 顧客目的、ターゲット、案件制約、確認できる事実に立ち戻る
8. 新しい案件を既存フォルダに上書きしない
9. すべての出力ファイルは、案件ごとの `SESSION_DIR/outputs/` 配下に保存する

## 作業フォルダルール

新しい案件、または新しい学習チャットを開始するたびに、必ず専用フォルダを作成する。

作業フォルダは、次の形式にする。

```text
sessions/YYYYMMDD_HHMM_案件名/
```

案件名がまだ分からない場合は、仮フォルダ名を使う。

```text
sessions/YYYYMMDD_HHMM_new_case/
```

以後、この作業フォルダを `SESSION_DIR` と呼ぶ。

新規案件開始時に、必ず以下の構成を作成する。

```text
SESSION_DIR/
  README.md
  assets/
  outputs/
```

出力ファイルを書くときは、必ず案件フォルダ配下のパスで書く。

例：

```text
SESSION_DIR/outputs/01_input_summary.md
SESSION_DIR/outputs/02_step1_facts.md
SESSION_DIR/outputs/03_step2_customer_goal.md
```

画像ファイルを参照するときも、案件フォルダ配下の `assets/` を使う。

例：

```text
SESSION_DIR/assets/product_01.jpg
SESSION_DIR/assets/scene_01.jpg
SESSION_DIR/assets/reference_01.png
SESSION_DIR/assets/preference_01.png
```

## 判断が難しい場合

判断できかねるものがある場合は、次の基準でAIが仮決定する。

1. 案件文に書かれている事実に沿っているか
2. ステップ3.5で作成したコンセプトに沿っているか
3. AIペルソナのニーズを満たせるか
4. ターゲットに伝わりやすいか
5. Canvaで実際に制作しやすいか
6. 案件の制約を守れるか

判断が難しい場合でも、処理を止めず、必ず次の形式で記録する。

```text
判断した内容：
判断根拠：
不確実な点：
なぜその判断を採用したか：
```

---

## 「次へ」の扱い

ユーザーが「次へ」と入力した場合は、現在の進行状況から次のステップに進む。

ただし、ステップ3以降は次の順番をノンストップで実行する。

1. ステップ3
2. ステップ3.5
3. ステップ4
4. ステップ5
5. ステップ6

ステップ3からステップ6の間では、ユーザーへの確認待ちで止めてはいけない。

---

## 最終確認

常に次の問いに戻る。

1. これは顧客の目的に合っているか
2. これはターゲットに伝わるか
3. 案件文のどの事実が根拠か
4. 自分の好みだけで決めていないか
5. AIペルソナの意見を事実として扱っていないか
6. ネガティブからポジティブへの変化が分かりやすいか
7. Canva上で実際に作れる形に翻訳できているか
8. 優先度1と優先度2の2案を出せているか
9. 判断に迷った場合、コンセプトとAIペルソナのニーズで判断できているか
10. 新しい案件を既存フォルダに上書きしていないか
