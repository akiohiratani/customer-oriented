# 案件情報整理

## 起動条件

ユーザーが案件情報を貼ったら実行する。

明示的に以下のような入力があった場合も実行する。

```text
案件情報入力
```

---

## 目的

ユーザーが貼り付けた案件情報を整理し、以降の分析に使える状態にする。

この段階では、まだ深い分析やデザイン判断をしない。

目的は、案件文・添付画像・希望イメージ・制約条件・未記載事項を分けて整理し、ステップ1以降で扱いやすい入力情報に変換することである。

---

## 実行内容

ユーザーが案件情報を貼ったら、以下を実行する。

1. 入力内容を整理する
2. 案件文から分かる事実を抽出する
3. 添付画像・参考画像から分かる事実を整理する
4. 未記載・未確認・要確認事項を整理する
5. `SESSION_DIR/outputs/01_input_summary.md` を作成する
6. `SESSION_DIR/README.md` の「案件情報入力」にチェックを入れる
7. チャットには要約だけ出す
8. 次に「ステップ1へ」と入力するよう案内する

---

## 出力ファイル

```text
SESSION_DIR/outputs/01_input_summary.md
```

---

## 整理時の注意

案件情報を整理するときは、以下を守る。

1. 案件文に書かれていることを優先する
2. 添付画像から読み取れることと、読み取れないことを分ける
3. 顧客目的をこの段階で断定しない
4. ターゲット心理をこの段階で断定しない
5. デザイン案をこの段階で作らない
6. 書かれていないことは補完せず、「未記載」「未確認」「要確認」とする
7. ユーザーの好みだけで情報を整理しない
8. 以降のステップで使いやすいように、事実を分類して保存する

## README.md の更新

`SESSION_DIR/README.md` の進行状況を更新する。

更新前：

```markdown
- [ ] 案件情報入力
```

更新後：

```markdown
- [x] 案件情報入力
```

## チャット出力形式

案件情報を整理した後、チャットには以下の形式で出力する。

```text
案件情報を整理しました。

作成ファイル：
- SESSION_DIR/outputs/01_input_summary.md

主な確認ポイント：
- 〇〇
- 〇〇
- 〇〇

未記載・要確認：
- 〇〇
- 〇〇

次は「ステップ1へ」と入力してください。
```

---

## チャットに出してはいけないこと

この段階では、チャットに以下を出してはいけない。

1. 詳細な分析全文
2. ターゲット心理の深い推論
3. AIペルソナ
4. 草案
5. Canva制作指示
6. 提出文
7. デザインの採用判断

詳細は必ず `SESSION_DIR/outputs/01_input_summary.md` に保存する。

---

## 例外対応

入力情報が不足している場合でも、処理は止めずに整理する。

不足している情報は、以下のように記録する。

```text
未記載：
未確認：
要確認：
```

案件情報がほとんどない場合は、チャットで次のように案内する。

```text
案件情報が不足しているため、現時点で分かる範囲だけ整理しました。

不足している情報は `SESSION_DIR/outputs/01_input_summary.md` に「未記載」「要確認」として記録しています。

次に進む場合は「ステップ1へ」と入力してください。
追加情報がある場合は、そのまま貼り付けてください。
```
